
# if文

## キーポイント

* if文を使うと「ある条件が正しい時だけ処理をする」というプログラムが書ける
* if文は次のように書く

  ```c#
  if (条件式) {
    処理
  }
  ```

* else句を使うと「条件式が正しくなかった時」の処理を書ける
* else句は次のように書く

  ```c#
  if (条件式) {
    処理A
  } else {
    処理B
  }
  ```

* 条件式が真のときは`true`、偽のときは`false`になる
* if文の中にif文を書いて、多重if文を作ることができる
* 論理演算子(`&&`や`||`)を使うと、複雑な条件をあらわすことができる

## 1 if文の基本

### 1.1 if文の書きかた

if文を使うと、ある条件が正しい時だけ処理をするというプログラムが書けるようになります。<br>
if文の書き方は次のようになります。

```c#
if (条件式)
{
  処理
}
```

「条件式」には、`==`などの「比較演算子」や、`&&`などの「論理演算子」を使って式を書きます。<br>
条件式が正しい時は、`{`から`}`の間の処理が実行されます。条件式が正しくないときは、この処理は飛ばされます。

次の例では、入力の値が`5`より小さければ「その数字は5より小さい」と出力した後「終了」と出力します。<br>
また、入力の値が`5`より小さくなければ(つまり`5`以上なら)「終了」とだけ出力します。

**コード**

```c#
Console.WriteLine("0～9の数字を入力してください");
string s = Console.ReadLine();
int x = int.Parse(s);

if (x < 5)
{
  Console.WriteLine("その数字は5より小さい");
}

Console.WriteLine("終了");
```

**実行例（１）**

```txt
0～9数字を入力してください>1
その数字は5より小さい
終了
```

**実行例（２）**

```txt
0～9数字を入力してください
5
終了
```

この例では、`Console.ReadLine`と`int.Parse`を使って、整数のデータを受け取っています。<br>
重要なのはその後です。

```c#
if (x < 5)
{
  Console.WriteLine("その数字は5より小さい");
}
```

この部分は、「もし`x < 5`（`x`が`5`より小さい）なら「その数字は5より小さい」と出力する」という意味になります。

`x`が`5`より小さくない(つまり`5`以上の)場合、`{`から`}`までのプログラムは無視されます。そのため、`5`以上の数値を入力すると「終了」とだけ出力されます。

このif文のように、何かの条件で処理が別れることを条件分岐といいます。<br>
また、「条件式が正しい」ことを「条件式が真(しん)」、「条件式が正しくない」ことを「条件式が偽(ぎ)」といいます。

### 1.2 比較演算子

条件式のところには、多くの場合、比較演算子(ひかくえんざんし)を使って条件を書きます。<br>
比較演算子は以下の6つです。

| 演算子 | 意味             |
|:------:|:-----------------|
| x == y | xとyは等しい     |
| x != y | xとyは等しくない |
| x > y  | xはyより大きい   |
| x < y  | xはyより小さい   |
| x >= y | xはy以上         |
| x <= y | xはy以下         |

「より大きい」「より小さい」を表す記号は数学と同じです。<br>
「等しい」は`=`を2つ繋いで`==`になります。「等しくない」は`!`が`=`の前につきます。<br>
「以上」「以下」は、数学では「≧」「≦」のように`=`を下につけますが、C#では`>=`,`<=`のように`=`を右側につけます。

次のプログラムは、入力された整数値がどんな条件を満たしているかを出力するプログラムです。

**コード**

```c#
Console.WriteLine("1～100の数字を入力してください";

int x = int.Parse(Console.ReadLine());

if (x < 10)
{
  Console.WriteLine("その数字は10より小さい");
}

if (x >= 20)
{
  Console.WriteLine("その数字は20以上");
}

if (x == 5)
{
  Console.WriteLine("その数字は5");
}

if (x != 100)
{
  Console.WriteLine("その数字は100ではない");
}

Console.WriteLine("終了");
```

**実行例（１）**

```txt
1～100の数字を入力してください
5
その数字は10より小さい
その数字は5
その数字は100ではない
終了
```

**実行例（２）**

```txt
1～100の数字を入力してください
100
その数字は20以上
終了
```

### 1.3 条件式に文字や文字列を使う

文字や文字列も、条件式に使うことができます。次のプログラムは、条件式に文字列を使う例です。

**コード**

```c#
string s = Console.ReadLine();

if (s == "fire")
{
  Console.WriteLine("火属性");
}
if (s != "wind")
{
  Console.WriteLine("風属性");
}
```

**実行例（１）**

```txt
fire
火属性
```

**実行例（２）**

```txt
wind
風属性
```

文字列の比較は「アルファベットの並び順」で行われます。これは「辞書順」とも呼ばれます。例えば、`'a'`と`'b'`では`'a'`のほうが小さい、`'y'`と`'z'`では`'z'`のほうが大きい、と判定されます。

また、大文字と小文字の比較では、常に大文字のほうが小さいと判定されます。つまり、`'Z' < 'a'`は真です。

比較は、文字列の先頭から1文字ずつ行われます。例えば、`"abc"`と`"acc"`では、`"abc"`のほうが小さいと判定されます。

1文字目はどちらも`'a'`で等しいですが、2文字目の`'b'`と`'c'`では`'b'`のほうが小さいからです。2文字目で大小関係が決まったので、3文字目以降は判定しません。

なお、文字列と数値を比較しようとするとエラーになります。`"123"`のように、数値のような文字列でもダメです。文字列を数値を比較するには、`int.Parse`や`.ToString`を使って、型をそろえる必要があります。

```c#
string s = "123";
int i = 123;
//if (s == i) {} // エラー. 文字列と数値は比較できない
//if (i == s) {} // エラー. 順番を逆にしても無意味
if (int.Parse(s) == i) {} // OK. 型を合わせるために s を int に変換
if (s == i.ToString()) {} // OK. 型を合わせるために i を string に変換
```

「1文字だけ調べたい」という場合は、文字列ではなく「文字」を比較します。以下は条件式に文字を使う例です。

**コード**

```c#
string s = Console.ReadLine();
char c = s[0];

if (c == 'a')
{
  Console.WriteLine("aです");
}
if (c >= 'A' && c <= 'Z')
{
  Console.WriteLine("大文字です");
}
```

**実行例（１）**

```txt
a
aです
```

**実行例（２）**

```txt
R
大文字です
```

文字列変数に続けて角括弧記号`[`と`]`を書くと、文字列から文字を取り出すことができます。取り出す文字の位置は、`[`と`]`の内側に書きます。最初の文字を「0番目」と数える点に注意してください(1番目ではありません)。

文字をデータとして表現するには、`'`(シングルクォーテーション、Shift+7キー)記号で文字を囲みます。

文字同士の比較は、文字列の場合と同じく辞書順です。

## 2 トランプゲームを作ろう(複雑なif文)

### 2.1 ブラックジャックのルール

if文の練習として、トランプゲームの「ブラックジャック」を作ります。ブラックジャックの基本ルールは次のように簡単なものです。

1. 手札の数字の合計が、21に一番近いプレイヤーの勝ち
2. ただし、合計が21より大きい場合は負ける
3. 絵札はすべて10として計算
4. 1のカードは1または11のどちらかを選べる

ただし、少し扱いの難しい「ルール4」は、最初は使わないことにします。

また、実際のブラックジャックでは、ジョーカーを除いた52枚のカードをシャッフルして使います。しかし、52枚のカードを管理するのは少々面倒です。そこで、カードを引くたびに乱数で1～13の値を決めるようにします。

ブラックジャックの進行手順(プログラム)は、以下の4つです。

1. 準備
   1. ディーラー(親)は、プレイヤーと自分に2枚ずつカードを配る
   2. ディーラーは、1枚目のカードだけを表にして見せる
2. プレイヤーのターン
   1. プレイヤーは、追加のカードを必要なだけ要求する
   2. 要求の結果、合計が22以上になったら、プレイヤーの負け
3. ディーラーのターン
   1. ディーラーは、合計が17以上になるまでカードを引き続ける
   2. ディーラーの合計が22以上になったら、ディーラーの負け
4. 勝敗判定
   1. 合計を比較し、21に近いほうが勝ち
   2. 合計が同じ場合は引き分け

それでは、以下の手順で、ブラックジャック用の新しいプロジェクトを作成しましょう。

1. Visual Studioを起動
2. 「新しいプロジェクトの作成」をクリック
3. 「コンソールアプリ」をクリック
4. 「プロジェクトの名前」を`BlackJack`にする
5. 「次へ」をクリック
6. 「作成」をクリック

### 2.2 プレイヤーにカードを配る

最初に、プレイヤーにカードを配る部分を作ります。`Program.cs`を開き、次のプログラムを追加してください。

```diff

+int playerNumber = 0; // プレイヤーのカードの合計
+
+// プレイヤーの1枚目
+int card = rand.Next(13) + 1; // カードを引く
+if (card >= 11)
+{
+  card = 10; // 11以上のカードは10として扱う
+}
+playerNumber += card; // 引いたカードの番号を合計に足す
```

勝敗を決めるために必要なのは、カードの合計だけです。そこで、合計値をあらわす`playerNumber`(プレイヤー・ナンバー)という変数を用意しました。

`card`(カード)変数は、引いたカードの番号をあらわします。カードを引く操作には`rand.Next`命令を使います。範囲に`13`を指定して`0`～`12`の数値を作り出し、`1`を足すことで、`1`～`13`というカード番号を生み出しています。

ただし、ブラックジャックでは絵札、つまり`11`以上のカードはすべて`10`として扱われます。そこで、if文を使って「引いたカードの数値が`11`以上場合、`10`にする」というプログラムを入れています。

続いて、2枚目のカードを引きましょう。プレイヤーの1枚目のカードを引くプログラムの下に、次のプログラムを追加してください。


### 2.4 プレイヤーのターンを作る

カードを配り終えたので、次はプレイヤーのターンを作ります。必要なだけカードを引きますが、何枚引くのかは事前には分かりません。そこで、for文を使って繰り返し処理を行います。それから、引く、やめるの判断に必要な情報も表示しましょう。

ディーラーが2枚目のカードを引くプログラムの下に、次のプログラムを追加してください。

```diff

 {
   card = 10; // 11以上のカードは10として扱う
 }
 dealerNumber += card; // 引いたカードの番号を合計に足す

+
+// プレイヤーのカード追加

+Console.WriteLine("あなたの合計：" + playerNumber);
+for (; ; )
+{
+  Console.WriteLine("ディーラーの1枚目：" + dealerFirstCard);

```

ここまでのプログラムが書けたら、Visual Sutidoウィンドウ上部の`>BlackJack`ボタンをクリックして、アプリを実行してください。プレイヤーとディーラーの合計と、プレイヤーの選択肢が表示されたら成功です。

というプログラムも、if文の基本的な使い方のひとつです。

### 2.5 プレイヤーターンの終了(else句)

else(エルス)句は、if文の後に書くことで「if文の条件が偽の時」、つまり「直前の条件が真でないとき」の処理を行えるようになります。<br>
else句の書き方は次のようになります。

```c#
if (条件式)
{
  処理1
}
else
{
  処理2
}
```

実際にelse句を使ってみましょう。プレイヤーが1を選択した場合のif文の下に、次のプログラムを追加してください。

```diff
   if (s == "1")
   {
     card = rand.Next(13) + 1; // カードを引く
     if (card >= 11)
     {
         card = 10; // 11以上のカードは10として扱う
     }
     playerNumber += card; // 引いたカードの番号を合計に足す
 
     Console.WriteLine("引いたカード：" + card);
     Console.WriteLine("あなたの合計：" + playerNumber);
     Thread.Sleep(1000); // 少し待つ
   }
+  else
+  {
+    break; // ターン終了
+  }
 } // プレイヤーのターンの終わり
```

このelse句はすぐ上にあるif文と連動しています。if文の条件が`false`になる、つまり条件が成立しない場合に、このelse句が実行されます。

### 2.9 勝敗判定(else if文)

else if(エルス・イフ)文は、else句に直接if文をつなげる書きかたです。「前のif文の条件が偽の場合に処理されるif文」となります。

else if文の書き方は次のようになります。

```c#
if (条件式1)
{
  処理1
}
else if (条件式2)

  処理2
}
```

### 2.10 Aの特殊処理を追加する(bool型)

ブラックジャックにおいて、A(エース)は`1`または`11`として扱われます。どちらにするかはプレイヤーが選択できます。ですが、基本的には機械的に決まります。「`11`にすると合計が`21`を越えてしまうなら`1`、越えないなら`11`」となります。

また、2枚以上のAを持っていても、`11`にできるのは1枚だけです。2枚以上`11`にすると合計が`21`を越えてしまうからです。

これらのことから、ゲームに必要な情報は「Aを持っているかどうか」だけで十分です。すべてのAは基本的に`1`として扱い、「`11`にしたほうが合計が良くなる」場合に限って`11`にします。

それでは、Aの有無をあらわす変数を追加しましょう。プレイヤーの変数を定義するプログラムに、次のプログラムを追加してください。

>`bool`型変数の名前には、`has`(ハズ)または`is`(イズ)で始まる名前を付けることが多いです。`has`は「～を持っている？」のように「所持、未所持」をあらわす変数に、`is`は「～ですか？」のように「状況」を表す場合に使います。<br>

### 2.12 エースの価値を決定する(多重if文)
